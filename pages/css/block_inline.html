블록 레벨 요소 ( block-level elements ) - 블록 레벨 요소는 (예: p요소 ) 블록 주 박스( block-level principal box )를 생성한다.display 속성 값의 block, list-item, table 중 어느 하나로 설정하면 요소는 블록 레벨 요소(block-levelboxes)로 변한다.block-level boxes는 BFC, block formatting context에 참여한다.
在 CSS 2.2 中，一个块级盒一般也是一个块容器盒 ( block container box ), 除非这个块级盒是一个表格盒 ( table box ) 或者一个替换元素 ( replaced element ) 的主盒。块容器盒 要么只包含块级盒，要么建立一个行内格式化上下文 ( IFC, inline formatting context )，此时这个块容器盒只包含行内级盒 ( inline-level boxes )。
任何直接包含在一个块容器元素中的文本 ( 即不在行内元素内 ) 必需被看做一个匿名行内盒。此类匿名行内盒从其块父亲块盒继承可继承的属性.非继承属性具有其初始值。在上面的例子中，匿名行内盒的 color 继承自 P，但是它的 background 是透明的。white-space 属性折叠起来的空白不会生成任何匿名行内盒。
list-item 该值使得一个元素 ( 例如 HTML 中的 LI )生成一个主要的块盒以及一个标记盒 ( marker box )。关于列表和列表的格式化的信息，参阅列表一节。
定位方式 ( Positioning schemes ) 在 CSS 2.2 中，一个盒子根据三种定位方式进行布局：
常规流 ( Normal flow ) 。常规流包含块级盒的块格式化和行内级盒的行内格式化，以及块级盒和行内级盒的相对定位。 浮动 ( Floats ) 。在浮动模型中，一个盒首先根据常规流进行布局，然后从流中取出并尽可能的向左或向右移动。内容流可能会在浮动元素周围。 绝对定位 ( Absolute positioning )。 在绝对定位模型中，一个盒完全从常规流中取出 ( 它对后续兄弟节点无影响 ) 然后相对于其包含块赋予一个位置。

### BFC, block formatting context 在一个块格式化上下文中，盒子垂直地一个接一个的进行布局，从包含块的开始处开始布局。两个兄弟盒子之间的垂直距离取决于他们的 margin 属性。一个块格式化上下文中的相邻的块级盒的垂直外边距和发生折叠 ( collapse )。
### IFC, inline formatting context 1. inline formatting context 由一个不包含block-level box的containing box建立。 2. box之间有水平方向的margin、border和padding。 3. 一个containg box形成一条线的矩形区域称为line box。
1. line box的高度总是足够包含其中的boxs。但是，它有可能会比其containing box中最高的box还要高 ( 例如，如果boxs通过对齐使得 baseline 上移 )。 2. 当一个box B 的高度小于包含它的line box的高度，B 垂直方向上的对齐由 vertical-align 决定。 当多个inline box不能水平的放进一个line box时，它们会分布在两个或更多的垂直方向以栈的方式堆叠的 ( vertically-stacked ) line box中。 3. line box在垂直方向上堆叠且没有垂直方向的分隔 ( 除非在其他地方指定 )，并且它们从来不会重叠。 4. 在同一个inline formatting context的line box通常高度不同 ( 例如，一行可能有一张很高的图片，另一行只包含文本 )。 当一行中的inline box的整体的宽度比包含它们的line box的宽度小时，他们在行盒中的水平分布取决于 text-align 属性。如果这个属性值为 justify，那么用户代理还应该会扩大inline box中的空格和单词 ( 在 inline-table 在 inline-block 中不会 )。 5. 如果一个inline box的宽度超过了line box的宽度，他们会分为多个盒子然后这些盒子会分布在多个line box中。如果一个inline box不能被分割 ( 例如：如果inline box包含一个字符，或者特定语言的分割规则不允许inline box的分割，又或者inline box受到 white-space 值为 nowarp 或者 pre 的影响 )，那么inline box会溢出行盒。 6. inline-level元素有可能被分割。
* line box在需要控制inline formatting context的行内级内容时被创建。一个line box如果没有文本，没有保留的空白符，没有margin、padding或border不为零的行内元素，没有其他in-flow的内容 ( 比如图像、inline-block或inline-table )，并且没有以一个
结束，那么这个行盒要被看作是一个用于确定其内部元素位置的高度为零的line box，对于其他情况，要看做它不存在。
### display속성
block 该值是一个元素生成一个主块盒 ( principal block box ).
inline-block 该值使得一个元素生成一个主要的行内级块容器 ( principal inline-level block container )。( inline-block 内部格式化为一个块盒，而该元素本身格式化为一个原子行内级盒 )。
inline 该值使得一个元素生成一个或多个行内盒。
list-item 该值使得一个元素 ( 例如 HTML 中的 LI )生成一个主要的块盒以及一个标记盒 ( marker box )。
none 该值使得元素不在格式化结构 ( formatting structure ) 中出现 ( 即在可视化媒体中，元素不产生任何盒子因此对布局没有影响 )。后代元素也不生成任何盒。元素及其内容将从格式化结构中完全删除，在子孙元素上设置’display’属性，不能覆盖此行为。
请注意，display 为 none 不会创建不可见的盒；它根本不会创建任何盒子。
